# 📘 기술적 구현 상세 내역 (Technical Specifications)

이 문서는 개인용 스마트 대시보드의 아키텍처와 핵심 기술 원리를 기록한 지식 자료입니다.

## 1. 핵심 기술 스택 (Core Tech Stack)
*   **React (Frontend)**: 사용자 인터페이스(UI)를 구성하는 핵심 엔진입니다. "컴포넌트(Component)" 기반으로 개발되어, 화면의 각 요소를 독립적인 부품처럼 관리합니다.
*   **Supabase (Backend as a Service)**: 데이터베이스(PostgreSQL)와 사용자 인증 기능을 제공합니다. 기존 `localStorage`의 한계를 넘어 실시간 데이터 보관을 가능하게 합니다.
*   **CSS Variables**: 테마와 디자인 색상을 전역 변수로 관리하여, 일관된 프리미엄 디자인을 유지합니다.

---

## 2. 데이터 흐름의 마법: `useSupabaseStore` (커스텀 훅)
이 프로젝트의 가장 핵심적인 기술적 성취는 **`useSupabaseStore`**라는 도구입니다.

### 💡 작동 원리
1.  **초기 로드 (Fetch)**: 사용자가 로그인하면 Supabase DB에서 해당 유저의 데이터를 가져와 화면(State)에 채웁니다.
2.  **낙관적 업데이트 (Optimistic Update)**: 사용자가 버튼을 누르면 DB 응답을 기다리지 않고 **화면을 즉시 업데이트**합니다. 이 덕분에 앱이 매우 빠르고 부드럽게 느껴집니다.
3.  **백그라운드 동기화**: 화면이 바뀐 직후, 백그라운드에서 Supabase API를 호출하여 DB의 값을 수정합니다.
4.  **타입별 지능적 처리**: 
    - **배열 데이터 (List)**: 할 일, 메모처럼 목록이 늘어나는 데이터를 처리합니다. 고유 ID를 비교해 추가/수정/삭제를 판단합니다.
    - **객체 데이터 (Map)**: 날짜별 런닝 기록, 일기처럼 '날짜'가 키(Key)가 되는 데이터를 관리합니다. 해당 날짜의 데이터만 콕 집어 `upsert`(없으면 넣고 있으면 수정) 합니다.

## 3. 스마트 기능의 논리 (Smart Logic)

### 🔄 할 일 자동 이월 (Todo Rollover)
- `useEffect`라는 리액트 도구를 사용하여, 오늘 날짜로 접속할 때마다 실행됩니다.
- 이전 데이터 중 `done: false`(미완료)인 항목을 찾아 현재 날짜의 리스트에 복사해 넣습니다. 이때 중복 방지를 위해 텍스트 내용을 비교하는 검증 과정을 거칩니다.

### 🖼️ 지능형 이미지 리사이징 (Canvas Engine)
- 용량이 큰 원본 사진을 그대로 DB에 넣으면 저장 실패나 속도 저하가 발생합니다.
- HTML5 Canvas를 활용하여 사진을 로드한 뒤, 특정 픽셀(최대 800px)로 그림을 다시 그린 후 압축된 데이터(JPEG)로 변환하여 저장합니다. 이를 통해 어떤 사진을 올려도 100KB 내외로 가볍게 저장됩니다.

## 4. 인증 보안 체계 (Auth & RLS)

### 🔐 Row Level Security (RLS)
- "내 데이터는 오직 나만 볼 수 있다"는 규칙을 데이터베이스 차원에서 강제하는 기술입니다.
- `auth.uid() = user_id`라는 정책을 통해, 코딩 실수로 다른 사람의 데이터가 노출되는 것을 원천 차단합니다.

### 👤 OAuth (Social Login)
- 구글 계정 연동을 통해 별도의 회원가입 없이도 안전하게 본인 인증을 수행합니다. Supabase Auth 가 세션을 관리하며, 브라우저가 꺼져도 로그인 상태를 유지해 줍니다.

- **CSS Grid & Flexbox**: 모든 카드는 내용의 길이에 따라 높이가 유동적으로 변하며, 아래의 카드들을 자연스럽게 밀어냅니다.
- **Glassmorphism**: 배경에 투명도와 블러(Blur) 효과를 주어 세련된 투명 유리 디자인을 구현했습니다.
- **Micro-interactions**: 버튼 호버(마우스 올리기), 체크박스 애니메이션 등을 통해 사용자에게 즉각적인 시각적 피드백을 줍니다.
