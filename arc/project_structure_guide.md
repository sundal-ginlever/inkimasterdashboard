# 📂 프로젝트 폴더 및 파일 가이드 (Project Structure Guide)

대시보드 프로젝트의 디렉토리 구조와 각 파일의 상세 역할입니다.

## 1. 루트 폴더 (Root Directory)
프로젝트의 가장 바깥쪽 폴더로, 설정 및 배포와 관련된 파일들이 위치합니다.
*   **`.env`**: Supabase URL이나 Anon Key 같은 민감한 비밀 정보를 담는 파일입니다. (보안상 매우 중요!)
*   **`package.json`**: 프로젝트 정체성(이름, 버전)과 사용된 라이브러리 목록을 정의합니다. `npm run start`나 `npm run build` 같은 명령어 설정이 들어있습니다.
*   **`vercel.json`**: Vercel 배포 시 서버 설정(라우팅 등)을 제어하는 설정 파일입니다.
*   **`public/`**: 빌드 과정 없이 그대로 사용되는 정적 파일(이미지, 아이콘 등)이 들어있는 폴더입니다.

---

## 2. `src/` 폴더: 모든 코딩의 핵심
우리가 직접 코딩하고 수정하는 모든 로직이 담긴 곳입니다.

### 🏠 메인 파일
*   **`App.jsx`**: 대시보드의 **심장(Root Component)**입니다. 모든 데이터의 상태(State)가 정의되어 있고, 각 탭/메뉴를 전환하는 로직이 들어있습니다.
*   **`App.css`**: 대시보드 전체의 디자인과 레이아웃(그리드, 폰트, 유리 효과 등)을 결정하는 스타일 시트입니다.
*   **`index.js`**: 리액트 앱을 실제 브라우저의 HTML에 연결해 주는 진입점입니다.

### 🧩 `components/` (UI 부품들)
*   **`auth/`**: 로그인 화면(`Auth.jsx`)을 담당하는 폴더입니다.
*   **`common/`**: 여러 화면에서 공통으로 쓰이는 부품(카드 형태, 팝업창, 스크롤 선택기 등)이 들어있습니다.
*   **`layout/`**: 화면의 뼈대인 상단 바(`Header.jsx`)와 사이드바 메뉴(`Sidebar.jsx`)를 관리합니다.
*   **`sections/`**: 각 탭(기록, 홈, 할 일 등) 대화면을 담당합니다.
    -   **`todo/`**: 할 일 탭 내부의 디테일한 요소(습관 트래커, 일기장, 타임로그 등)가 별도로 분리되어 있습니다.

### ⚙️ `hooks/`, `lib/`, `utils/` (로직 도구들)
*   **`hooks/useSupabaseStore.js`**: DB와 통신하며 데이터를 가져오고 저장하는 **커스텀 도구**입니다. 이 프로젝트의 가장 복잡하고 핵심적인 로직이 담겨 있습니다.
*   **`lib/supabase.js`**: Supabase를 사용하기 위한 접속 설정을 담당합니다.
*   **`utils/`**: 날짜 계산(`index.js`)이나 이미지 크기 조절(`imageResizer.js`) 같은 **도우미 함수**들을 모아둔 곳입니다.
*   **`constants/index.js`**: 앱 전역에서 공통으로 쓰는 값(기본 날씨 설정, 탭 이름 등)을 정의하여 코드의 재사용성을 높입니다.

---

## 💡 요약: 파일들이 일하는 방식
1.  **사용자가 입장**하면 `index.js`가 `App.jsx`를 실행합니다.
2.  `App.jsx`는 `useSupabaseStore`를 통해 DB에서 데이터를 불러옵니다.
3.  불러온 데이터를 `TodoSection.jsx`나 `HabitTracker.jsx` 같은 하위 컴포넌트들에게 전달합니다.
4.  각 컴포넌트는 전달받은 데이터를 `App.css`에 정의된 스타일대로 예쁘게 화면에 그려줍니다.
