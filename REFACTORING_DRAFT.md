# Dashboard MVP 구조 개편(리팩토링) 계획서

이 문서에서는 기존의 단일 파일(`dashboard.jsx`, 총 1,200여 줄) 형식으로 작성된 MVP 수준의 대시보드 코드를, 장기적 유지보수와 기능 확장을 위해 깔끔하게 재구성(리팩토링)하는 방향성을 정리합니다. 다음 작업 시 이 계획서를 바탕으로 작업을 시작할 예정입니다.

## 1. 컴포넌트 분리 (Component Modularization)
현재 `App.jsx` 파일에 모든 UI(날씨, 차트, 메모, 모달 등)가 몰려 있습니다. 이를 역할별로 쪼개어 독립적인 컴포넌트로 관리합니다.
- `src/components/WeatherWidget.jsx`: 실시간 날씨 표시 UI 담당
- `src/components/MemoPad.jsx`: 터치/드래그 가능한 메모 리스트 담당
- `src/components/RunningChart.jsx`: 러닝 기록 막대그래프 시각화 담당
- `src/components/RunLogModal.jsx`: 스크롤 휠 기반 자물쇠형 러닝 기록 입력 모달
- `src/components/Dashboard.jsx`: 위 컴포넌트들을 조합하는 중심 레이아웃

## 2. 상태 관리 및 비즈니스 로직 분리 (Custom Hooks 도입)
`App.jsx` 내부에 뒤섞인 브라우저 저장소 연동(`localStorage`), 위젯 상태 관리(`useState`), 생명주기(`useEffect`) 로직을 추출합니다.
- **`useWeather` 훅**: 날씨 등 외부 데이터를 불러오는 로직 추출.
- **`useMemos` 훅**: 메모 데이터 로드, 드래그 스왑 정렬, 저장 로직 추출.
- **`useRunLogs` 훅**: 러닝 데이터 기록, 누적 차트 연산, 기본 이미지 적용 등 로직 추출.

## 3. 통신 및 데이터 계층 분리 (Services/API 계층 추출)
컴포넌트 내에 작성된 외부 세계와의 비동기 통신 로직을 모듈화합니다.
- `src/services/weatherApi.js` (또는 `src/api/...`): Open-Meteo 날씨 API 데이터 조회 및 현재 위치 좌표 연산 로직만 담고 있는 통신 모듈로 독립.
- 추후 주식, 환율 등 다른 API가 연동될 때를 대비해 확장성 고려.

## 4. 스타일 체계 구조화 (CSS 개편)
현재 전역 CSS(`index.css`)나 인라인 스타일(`style={{...}}`)로 구성된 방식에서 모던 리액트 스타일링 방식으로 전환합니다.
- 클래스 이름 충돌을 피하기 위해 **CSS Modules**를 도입하거나, 직관적인 **Tailwind CSS**를 사용하여 유지보수를 개선합니다.

## 추가개선사항 기재
1. 홈탭 메모 드래그 기능은 원상복귀하여 드래그 기능을 삭제하자, 스와이프 기능과 중첩됨.
2. 러닝거리 5자리의 디자인이 위에 표시되는 시간과 좌우 간격이 유사하도록 숫자간 거리를 넓히자.
3. 자산, 가계부, 독서 탭의 추가버튼으로 기록이 저장이 되지 않는다. 자산, 독서의 추가버튼 탭은 삭제하자
4. 가계부는 해당 탭에서 추가 기록이 가능하도록 추가버튼 내 저장버튼을 활성화하자
5. 자산, 독서 탭의 신규기록은 되지 않더라도 기록된 내용에 대한 수정은 해당 탭에서 가능하도록 하자
